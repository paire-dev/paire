# Paire

Paire is a new layer for helping humans understand code generated by AI agents.

It turns changes into structured review state through first-class commands and opens a small local UI so a human can quickly inspect what changed.

No hosted service. No LLM built into Paire. Just Git, local state, exported packets, and a browser review UI.

> [!NOTE]
>
> 100% secure and local first

## Install

```sh
curl -fsSL https://raw.githubusercontent.com/paire-dev/paire/main/scripts/install.sh | bash
```

The release binary does not require Bun. Paire installs to `~/.local/bin/paire` by default and links `paire` into `/usr/local/bin` when possible.

To inspect the installer first:

```sh
curl -fsSLo /tmp/paire-install.sh \
  https://raw.githubusercontent.com/paire-dev/paire/main/scripts/install.sh
less /tmp/paire-install.sh
bash /tmp/paire-install.sh
```

## Start

Run this once in a Git repo:

```sh
paire start --base main --goal "Review current changes"
paire install
```

`paire install` adds agent instructions to files such as `AGENTS.md` or `CLAUDE.md`, so your coding agent knows to run Paire before it finishes.

## Use

Run:

```sh
paire it
```

If your worktree is dirty, Paire reviews the uncommitted diff. If it is clean, Paire reviews committed changes since the last finalized review.

Use the commands Paire prints:

1. Inspect the read-only context with `paire review context`.
2. Add or update claims with `paire claim add`, `paire claim edit`, and `paire claim evidence add/remove`.
3. Acknowledge generated or mechanical files with `paire file acknowledge`.
4. Finish with `paire review finalize`.

The Review UI runs on `127.0.0.1` and shows the reflected review state for the current branch or selected review.

## Commands

```txt
paire start --base <ref> --goal <text>
paire install
paire it
paire review
paire review context [--review <id>|--diff <base..head>] [--json]
paire review list [--limit 20] [--json]
paire review open <review-id>
paire review finalize
paire claim add --title <text> --importance <level> --thread-id <id> --before <text|null> --after <text|null> --evidence <path:start-end:change>
paire claim edit --claim <id> [--work-status <status>]
paire claim list [--json]
paire claim show --claim <id> [--json]
paire claim evidence add --claim <id> --evidence <path:start-end:change>
paire claim evidence remove --claim <id> --evidence <evidence-id>
paire file acknowledge --path <path> --reason <text>
paire status
paire sync
paire reset
paire server start [--no-open]
paire server stop
paire upgrade [--force]
paire version
```

`paire upgrade` reinstalls the latest release (re-running the install script). `paire it` also checks once a day and prints an upgrade hint when a newer version is available; set `PAIRE_NO_UPGRADE_CHECK=1` to disable that check.

## Local State

Paire stores local state under `~/.paire` by default:

```txt
~/.paire/paire.db
~/.paire/artifacts/
~/.paire/projects/<project-key>/current-context.json
~/.paire/projects/<project-key>/annotated-diff.txt
```

Override this with `PAIRE_HOME`.

Sessions are per Git branch, while concrete reviews are keyed by resolved Git targets. Linked Git worktrees get separate project paths, so context artifacts stay isolated.

## Development

Requires [Bun](https://bun.sh).

```sh
bun install
bun src/cli.ts --help
bun run typecheck
bun test
bun run smoke
```

Build the local binary:

```sh
bun run build
```
