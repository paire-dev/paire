# Paire CLI

Paire is a new layer for helping humans understand code generated by AI agents.

It turns committed Git changes into a structured review packet, asks the agent to write review JSON, applies that review state, and opens a small local UI so a human can quickly inspect what changed and what the agent claimed.

No hosted service. No LLM built into Paire. Just Git, local state, exported packets, and a browser review UI.

> [!NOTE]
>
> 100% secure and local first

## Install

```sh
curl -fsSL https://raw.githubusercontent.com/paire-dev/paire/main/scripts/install.sh | bash
```

The release binary does not require Bun. Paire installs to `~/.local/bin/paire` by default and links `paire` into `/usr/local/bin` when possible.

To inspect the installer first:

```sh
curl -fsSLo /tmp/paire-install.sh \
  https://raw.githubusercontent.com/paire-dev/paire/main/scripts/install.sh
less /tmp/paire-install.sh
bash /tmp/paire-install.sh
```

## Start

Run this once in a Git repo:

```sh
paire start --base main --goal "Review current changes"
paire install
```

`paire install` adds agent instructions to files such as `AGENTS.md` or `CLAUDE.md`, so your coding agent knows to run Paire before it finishes.

## Use

Commit your work, then run:

```sh
paire it
```

Paire reviews committed code only. If your worktree is dirty, it will ask you to commit or stash first.

When Paire prints **Action required**, follow the printed steps:

1. Inspect the Git diff.
2. Read the exported packet JSON.
3. Write the review result JSON.
4. Apply it with `paire review --apply <file>`.
5. Open the local Review UI URL when printed.

The Review UI runs on `127.0.0.1` and shows the applied review state for the current branch.

## Commands

```txt
paire start --base <ref> --goal <text>
paire install
paire it
paire review
paire review --apply <file>
paire review --stdin
paire status
paire sync
paire reset
paire server start [--no-open]
paire server stop
paire version
```

## Local State

Paire stores local state under `~/.paire` by default:

```txt
~/.paire/paire.db
~/.paire/artifacts/
~/.paire/projects/<project-key>/current-packet.json
```

Override this with `PAIRE_HOME`.

Sessions are per Git branch. Linked Git worktrees get separate project paths, so review packets and result files stay isolated.

## Development

Requires [Bun](https://bun.sh).

```sh
bun install
bun src/cli.ts --help
bun run typecheck
bun test
bun run smoke
```

Build the local binary:

```sh
bun run build
```
