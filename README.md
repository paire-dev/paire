# Paire

Paire is a new layer for helping humans understand code generated by AI agents.

It turns changes into a structured review draft, asks the agent to create a high-level review, and opens a small local UI so a human can quickly inspect what changed.

No hosted service. No LLM built into Paire. Just Git, local state, exported packets, and a browser review UI.

> [!NOTE]
>
> 100% secure and local first

## Install

```sh
curl -fsSL https://raw.githubusercontent.com/paire-dev/paire/main/scripts/install.sh | bash
```

The release binary does not require Bun. Paire installs to `~/.local/bin/paire` by default and links `paire` into `/usr/local/bin` when possible.

To inspect the installer first:

```sh
curl -fsSLo /tmp/paire-install.sh \
  https://raw.githubusercontent.com/paire-dev/paire/main/scripts/install.sh
less /tmp/paire-install.sh
bash /tmp/paire-install.sh
```

## Start

Run this once in a Git repo:

```sh
paire start --base main --goal "Review current changes"
paire install
```

`paire install` adds agent instructions to files such as `AGENTS.md` or `CLAUDE.md`, so your coding agent knows to run Paire before it finishes.

## Use

Run:

```sh
paire it
```

If your worktree is dirty, Paire creates a worktree review draft. If it is clean, Paire reviews committed changes since the last applied revision.

When Paire prints **ACTION_REQUIRED**, follow the printed steps:

1. Inspect the diff Paire names.
2. Edit the generated draft in place.
3. Run the apply command Paire prints until it exits 0.

On success, Paire opens the Review UI automatically. The Review UI runs on `127.0.0.1` and shows the applied review state for the current branch.

## Commands

```txt
paire start --base <ref> --goal <text>
paire install
paire it
paire review
paire review --apply <file>
paire review --check <file>
paire review --stdin
paire worktree --apply <file>
paire worktree --check <file>
paire status
paire sync
paire reset
paire server start [--no-open]
paire server stop
paire upgrade [--force]
paire version
```

`paire upgrade` reinstalls the latest release (re-running the install script). `paire it` also checks once a day and prints an upgrade hint when a newer version is available; set `PAIRE_NO_UPGRADE_CHECK=1` to disable that check.

## Local State

Paire stores local state under `~/.paire` by default:

```txt
~/.paire/paire.db
~/.paire/artifacts/
~/.paire/projects/<project-key>/current-packet.json
~/.paire/projects/<project-key>/review-draft.json
```

Override this with `PAIRE_HOME`.

Sessions are per Git branch. Linked Git worktrees get separate project paths, so review packets and draft files stay isolated.

## Development

Requires [Bun](https://bun.sh).

```sh
bun install
bun src/cli.ts --help
bun run typecheck
bun test
bun run smoke
```

Build the local binary:

```sh
bun run build
```
