Agent-first CLI

paire-cli

A Bun-powered command-line tool that gathers Git context and produces structured output for AI coding agents — briefs for PRs, impact reviews, and hook automation.

Bun runtime No LLM calls --json everywhere Stable exit codes
Overview CLI Router paire impact Brief pipeline Git layer Config Agent-first

What it does

paire-cli sits between your Git repo and AI agents. It never calls an LLM itself — instead it collects diff context, formats prompts, and writes machine-readable artifacts agents consume.

LOCAL

paire it

Renders .paire/brief.html from branch diff + recent commits. Opens in browser on macOS/Linux.

PR

paire push

Posts the same brief as Markdown to the current GitHub PR via gh pr edit.

AGENT

paire impact

Prints a self-contained prompt to stdout. The agent writes .paire/impact.md with product-impact cards.

CLI router

src/cli.ts is a thin dispatcher. Global flags are parsed first; the first non-flag argument becomes the command name. 

flowchart TD
  A["process.argv"] --> B{Global flags?}
  B -->|--help / --version| C["Print & exit 0"]
  B -->|otherwise| D["Resolve command name"]
  D --> E{Known command?}
  E -->|no| F["CliError → exit 2"]
  E -->|yes| G["Build GlobalOptions\n--json, -y, --answers-file, cwd"]
  G --> H["command.handler(args, options, answers)"]
  H --> I["printResult(result, options)"]
  I --> J{exit code}
  J -->|0| K["Success"]
  J -->|1–4| L["Typed failure"]

paire impact — the agent loop

The CLI gathers PR context and emits a structured prompt. The AI agent (Cursor, Claude Code, Codex…) reads stdout, analyzes the embedded diffs, and writes .paire/impact.md. No shell commands needed after that.

ImpactContext shape

branch: string
baseRef: string
title: first commit subject
commitSubjects: string[]
diffStat: string
files: ImpactFile[]
truncatedFileCount: number

Prompt structure (renderImpactPrompt)

  1. Task header — write to output path, stop
  2. Phase 1 — identify impact areas (UX, API, …)
  3. Phase 2 — 1–4 items per area with Before/After
  4. Confidence — observed / inferred / unknown
  5. Output template — <details> cards
  6. PR context — commits, stat, file manifest, patches
impact.ts command flags
--base=branch override diff base
--output=path where agent writes (default .paire/impact.md)
--prompt-out=path also persist prompt to disk
--json wrap metadata + message in JSON

Brief object

Lightweight snapshot — title, branch, diff stat, recent commits, ISO timestamp. HTML wraps the Markdown in a styled <pre> block.

git diff --stat ${baseRef}...HEAD

Git layer

All Git operations go through git(args, cwd) using Bun's shell (Bun.$). Failures throw CliError with exit code 4. 

flowchart TD
  START["baseBranch(cwd)"] --> UP{"Upstream exists &\ndiffers from current?"}
  UP -->|yes| RET1["return upstream\n(e.g. origin/main)"]
  UP -->|no| LOOP["Try candidates in order"]
  LOOP --> C1["origin/main"]
  C1 -->|missing| C2["origin/master"]
  C2 -->|missing| C3["main"]
  C3 -->|missing| C4["master"]
  C4 -->|missing| ERR["CliError exit 4"]
  C1 & C2 & C3 & C4 -->|found| RET2["return candidate"]

  style START fill:#151b23,stroke:#34d399
  style ERR fill:#450a0a,stroke:#ef4444

Three-dot diff (base...HEAD) compares from the merge-base of base and HEAD to HEAD — the standard PR diff range. Config baseBranch overrides auto-detection for teams using non-standard bases like develop.

  includeHistory: true

Agent-first design principles

Every command is built so CI pipelines and coding agents can invoke it without interactive prompts.

📦

Machine-readable output

--json on every command. printResult() wraps { ok, message, ...data }.

📝

Bulk answers

--answers-file path.json feeds prompt answers without TTY interaction.

🎨

TTY-aware

Color disabled when stdout is not a TTY or NO_COLOR is set.

🔒

No env required

Works on first run. paire doctor tolerates missing $HOME.

Typical agent workflow (AGENTS.md)

  1. Before pushing → run paire it or paire push
  2. For PR impact review → run paire impact
  3. Read the stdout prompt — do not run other commands
  4. Write the rendered Markdown to .paire/impact.md
